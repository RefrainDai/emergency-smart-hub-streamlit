# emergency_agent 演示接入层

本目录用于将师兄提供的 `emergency_agent` 智能体模块以演示模式接入应急智枢平台。

## 定位

该目录不是资源缓存预测主线的替代模块，而是“多模态智能体任务服务来源”的轻量接入层。平台通过演示智能体工具调用轨迹，将灾害图像、文本请求等多模态处理过程转化为资源缓存预测所需的任务负载。

## 默认行为

- 默认不加载 Qwen3-VL、torch、transformers、PSPNet 或任何大模型权重。
- 默认不执行真实图像分割、真实视觉问答或真实报告生成推理。
- 默认使用固定演示工具调用轨迹，便于无模型环境直接演示。

## 文件说明

- `agent_schema.py`：定义智能体工具调用事件结构。
- `mock_agent_runner.py`：生成不依赖大模型的演示工具调用轨迹。
- `agent_event_mapper.py`：将 `tool_history` 转换为平台可用的任务负载表。
- `sample_agent_outputs.json`：固定演示工具调用轨迹。

## tool_history 到任务负载

映射逻辑如下：

| 工具 | 任务类型 | 资源权重 |
| --- | --- | --- |
| `vqa` / `VQA快速问答` | 快速问答业务 | 1 |
| `rescuenet_segmentation` | 语义分割业务 | 3 |
| `image_analysis` | 灾害图像深度分析业务 | 4 |
| `report_making` | 结构化报告生成业务 | 2 |
| `tool_ending` | 结果回传业务 | 0.5 |

`thinking_and_plan` 是轻量规划步骤，演示模式中映射为任务规划业务，资源权重为 0.5。

请求到达量由资源权重与工具调用次数生成，仅表示演示负载，不代表真实无人机采集数据或真实云资源监控数据。

## 后续扩展

后续如需接入真实 Qwen3-VL 智能体，可在高级模式中：

1. 独立安装大模型依赖和权重；
2. 在外部服务中运行真实智能体；
3. 将真实工具调用结果整理为本目录定义的 `tool_history` 结构；
4. 再通过 `agent_event_mapper.py` 转换为任务负载。

建议保持真实模型服务与资源缓存预测平台解耦，避免影响平台启动速度和交付稳定性。
