# 数据格式与页面参数说明

本文档说明平台上传文件的格式要求，以及各页面可调参数的含义。面向演示和评审使用，字段名仅用于文件读取兼容；平台界面会以中文术语展示。

## 1. 任务负载文件

对应页面：**灾害任务接入**。

任务负载文件用于描述灾害响应过程中不同智能服务在各时间步的请求到达情况。平台支持 `CSV`、`JSON`、`JSONL` 三类格式，推荐使用 `CSV`。

最小字段要求如下：

| 内部字段名 | 界面显示 | 含义 | 示例 |
| --- | --- | --- | --- |
| `log_time` | 时间步 | 请求发生的时间，可为日期时间或可解析时间字符串 | `2026-04-01 00:00:00` |
| `num` | 请求到达量 | 当前时间步到达的任务请求数量 | `18` |
| `access_key_id` | 请求序列 | 一条连续灾害事件或业务流的序列标识 | `地震救援序列001` |
| `game_name` | 任务类型 | 多模态任务服务类型 | `受灾目标识别` |

平台也支持部分别名：

| 可用别名 | 自动识别为 |
| --- | --- |
| `timestamp`、`time`、`ts` | `log_time` |
| `arrivals`、`arrival`、`current_arrivals`、`request_count`、`requests` | `num` |
| `sequence_id`、`seq_id`、`user_id` | `access_key_id` |
| `service_type`、`business_type`、`task_type` | `game_name` |

任务负载文件只需要描述请求到达情况。空闲算力资源、等待服务队列等运行状态会在平台演示过程中根据当前参数与决策过程逐步生成。

## 2. 历史决策轨迹文件

对应页面：**历史案例库**。

历史决策轨迹用于构建“历史案例库”。推荐格式为 `JSONL`，即每一行是一个完整 JSON 记录。每条记录代表一个历史时间步，平台会从连续轨迹中切分出案例。

推荐字段如下：

| 字段 | 界面显示 | 含义 |
| --- | --- | --- |
| `t` | 历史时隙 | 轨迹内的时间步编号 |
| `game_name` | 任务类型 | 历史任务服务类型 |
| `access_key_id` | 请求序列 | 历史请求序列标识 |
| `state_summary.resource_queue` | 空闲算力资源 | 当前时隙可用服务能力 |
| `state_summary.waiting_queue` | 等待服务队列 | 已到达但尚未完成服务的请求量 |
| `state_summary.current_arrivals` | 请求到达量 | 当前时隙新增请求量 |
| `state_summary.baseline_action` | 即时需求缺口 | 仅根据当前缺口计算的即时需求 |
| `raw_action` | 缓存调整量 | 相对即时需求的调节方向与幅度 |
| `allocation` | 资源缓存量 | 当前时隙建议预留或预热的资源量 |
| `reward` | 综合评价值 | 结合保障效果和资源冗余得到的评价 |
| `hit_rate` | 请求保障率 | 当前决策满足请求的比例 |
| `waste_rate` | 资源冗余率 | 当前决策中未被有效使用的资源比例 |

平台会把历史轨迹按“任务类型 + 请求序列”分组，再根据最近负载窗口、当前状态、历史缓存决策和后续反馈评价构建案例。高质量案例会优先作为 ICL 缓存决策的上下文参考。

## 3. 灾害任务接入页面参数

| 参数 | 含义 | 作用 |
| --- | --- | --- |
| 任务负载来源 | 选择内置演示数据、上传文件或读取本机文件 | 决定平台本次演示使用哪份请求负载 |
| 上传任务负载文件 | 上传 `CSV`、`JSON` 或 `JSONL` 文件 | 用自定义灾害任务负载替换内置样例 |
| 本机数据文件路径 | 从本机路径读取任务负载文件 | 适合调试或队内演示，不建议比赛现场手动输入复杂路径 |

## 4. 历史案例库页面参数

| 参数 | 含义 | 作用 |
| --- | --- | --- |
| 历史决策轨迹来源 | 选择内置历史轨迹、上传轨迹或读取本机轨迹 | 决定案例库从哪份历史决策数据构建 |
| 最多读取记录数 | 限制读取的历史轨迹行数，`0` 表示不限制 | 用于控制构建速度和案例库规模 |
| 最近负载窗口长度 | 构建案例时回看多少个历史时隙 | 窗口越长，案例包含的趋势信息越充分，但对轨迹长度要求更高 |
| 反馈评价观察窗口 | 评估一个历史决策后续影响的时间范围 | 用于计算案例质量和反馈评价 |
| 最大案例数 | 限制最终进入案例库的案例数量，`0` 表示不限制 | 控制检索规模，案例过多时可提高页面响应速度 |

## 5. 相似案例检索页面参数

| 参数 | 含义 | 作用 |
| --- | --- | --- |
| 选择任务请求序列 | 选择当前要分析的灾害任务序列 | 平台会基于该序列当前状态检索相似历史案例 |
| 最近负载窗口长度 | 当前检索时回看的近期请求窗口 | 影响负载趋势相似度计算 |
| 选择当前时隙 | 指定当前决策发生在哪个时间步 | 不同时间步的请求量、等待队列和资源状态不同 |
| Top-K | 返回的相似历史案例数量 | 数值越大，参考案例越多；数值过大可能引入弱相关案例 |

检索相似度会综合当前状态、请求趋势、负载状态类型、任务类型匹配和案例质量。

## 6. ICL缓存决策页面参数

| 参数 | 含义 | 作用 |
| --- | --- | --- |
| 方法 | 选择即时规则分配基线、启发式状态调节基线、相似案例缓存预测或 ICL少样本缓存决策 | 决定本次资源缓存量的计算方式 |
| 最近负载窗口 | 预测时回看的请求负载长度 | 用于形成当前上下文状态 |
| Top-K | ICL 决策参考的相似案例数量 | 决定上下文中提供多少历史案例 |
| 每条序列最多步数 | 每条任务序列最多运行多少个时间步 | 控制演示运行时间 |
| 等待惩罚系数 | 等待服务队列对综合评价的惩罚强度 | 数值越大，平台越重视降低排队等待 |
| 冗余惩罚系数 | 空闲或浪费资源对综合评价的惩罚强度 | 数值越大，平台越重视减少资源冗余 |
| 初始空闲算力资源 | 每条序列开始时的可用资源量 | 影响前几个时间步的等待和缓存结果 |
| 初始等待服务队列 | 每条序列开始时已积压的请求量 | 用于模拟灾害响应开始时的排队压力 |
| 缓存调整尺度 | 将缓存调整量转换为实际资源量的尺度 | 数值越大，同样的调整幅度对应更大的资源变化 |
| 单时隙最大资源缓存量 | 单个时间步允许建议的最大缓存量 | 防止极端请求导致过大的资源预留建议 |
| ICL推理模型接口（高级配置） | 外部模型接口设置 | 不配置也可以运行；未配置时使用本地案例检索与规则化推理完成演示 |

## 7. 调度效果评估页面参数

| 参数 | 含义 | 作用 |
| --- | --- | --- |
| 每条序列对比步数 | 策略对比时每条任务序列运行的时间步数量 | 用于平衡评估覆盖范围和运行速度 |
| 空闲算力资源 | 单状态敏感性分析中的当前可用资源 | 观察资源充足或不足时不同缓存量的效果 |
| 等待服务队列 | 单状态敏感性分析中的当前积压请求 | 观察已有排队压力对决策的影响 |
| 当前请求到达量 | 单状态敏感性分析中的新增请求量 | 观察突发请求到达时的资源需求变化 |
| 单时隙最大资源缓存量 | 单状态分析中的缓存上限 | 约束建议资源量范围 |
| 缓存调整尺度 | 单状态分析中的调整量尺度 | 影响曲线中资源缓存量变化幅度 |
| 等待惩罚系数 | 单状态分析中的等待惩罚 | 体现保障请求优先级 |
| 冗余惩罚系数 | 单状态分析中的资源冗余惩罚 | 体现资源节约优先级 |
| 评价尺度 | 综合评价值的缩放参数 | 便于观察不同配置下评价曲线变化 |

## 8. 单步可解释分析页面参数

| 参数 | 含义 | 作用 |
| --- | --- | --- |
| 选择时隙 | 选择已运行结果中的某一个时间步 | 查看该步的当前状态、资源缓存量、决策来源、相似案例和反馈评价 |

## 9. 结果导出页面按钮

| 按钮 | 导出内容 |
| --- | --- |
| 导出运行结果 | 当前预测运行的逐步结果 |
| 导出策略对比结果 | 调度效果评估页生成的策略对比表 |
| 导出单步决策解释 | 当前运行结果中每个时间步的解释表 |
| 导出历史案例表 | 当前案例库中的案例摘要 |
| 生成演示报告摘要 | 基于当前运行数据生成 Markdown 摘要 |

导出内容均基于当前平台加载的数据、当前参数和本次运行结果生成，不写入固定结论。
