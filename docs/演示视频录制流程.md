# 演示视频录制流程

## 1. 推荐视频时长

建议控制在 2 到 3 分钟。重点展示系统定位、核心流程和一次完整演示，不需要逐项解释所有参数。

## 2. 录制前准备

1. 打开平台并确认页面可以访问。
2. 推荐使用浏览器全屏或 125% 以下缩放，避免图表拥挤。
3. 进入“多模态任务接入”页面，确认演示智能体工具调用轨迹可生成。
4. 进入“灾害任务接入”页面，确认内置任务负载可正常加载。
5. 进入“历史案例库”页面，提前构建一次历史案例库。
6. 进入“ICL缓存决策”页面，提前运行一次预测。
7. 进入“调度效果评估”页面，提前生成策略对比。
8. 准备录屏软件，确认麦克风和系统声音正常。

## 3. 推荐页面顺序

```text
系统总览
→ 多模态任务接入
→ 灾害任务接入
→ 历史案例库
→ 相似案例检索
→ ICL缓存决策
→ 调度效果评估
→ 单步可解释分析
→ 结果导出
```

## 4. 每页建议讲解词

### 系统总览

“这是 O₂Cache 团队的应急智枢系统，面向灾害响应场景下的多模态智能服务请求。平台展示如何通过历史案例库、相似案例检索和上下文学习机制预测下一时隙资源缓存量，为模型实例预热、GPU/显存预留和并发资源分配提供辅助决策。”

### 多模态任务接入

“本页展示演示智能体如何把灾害图像与文本请求拆分为任务规划、语义分割、图像分析、报告生成和结果回传等工具调用，并进一步转化为资源缓存预测所需的多类型服务负载。”

### 灾害任务接入

“这里展示灾害场景下不同多模态服务的请求负载，包括快速问答、受灾目标识别、结构化报告生成和灾害图像深度分析。平台会把请求到达量整理为连续任务序列，作为后续资源缓存预测的输入。”

### 历史案例库

“历史案例由历史状态、请求窗口、缓存决策和反馈评价组成。高质量案例会成为 ICL 决策的上下文参考，帮助当前时隙找到相近场景下的资源配置经验。”

### 相似案例检索

“当前页面展示 Top-K 相似历史案例。检索时会同时考虑任务类型、近期请求趋势、空闲算力资源、等待服务队列和案例质量，因此这些案例可以作为当前决策依据。”

### ICL缓存决策

“这里展示当前状态、Top-K 相似案例和上下文输入模板如何共同形成资源缓存量预测。未配置外部 ICL 推理模型时，平台仍可通过本地案例检索与规则化推理完成演示。”

### 调度效果评估

“本页对比不同策略在当前演示数据上的请求保障率、资源冗余率和综合评价值。结果只基于当前数据，不做超出测试范围的外推。”

### 单步可解释分析

“这里可以复盘某一个时隙的决策过程，包括当前请求状态、资源缓存量、相似历史案例、决策来源和反馈评价，便于解释为什么产生这个资源建议。”

### 结果导出

“最后可以导出运行结果、策略对比结果、单步决策解释、历史案例表，并生成演示报告摘要，方便整理答辩材料。”

## 5. 录制注意事项

- 不要展示访问密钥、个人路径或无关桌面内容。
- 录制前先完成案例库构建和一次预测，减少等待时间。
- 每页停留 10 到 20 秒即可，重点讲清楚主流程。
- 若页面刷新后状态丢失，按推荐流程重新运行一遍。
- 视频中不要称平台已经接入真实无人机控制或真实云资源控制接口。

## 6. 如何导出演示报告摘要

1. 完成一次 ICL 缓存决策。
2. 在“调度效果评估”页面生成策略对比。
3. 进入“结果导出”页面。
4. 点击“生成演示报告摘要”。
5. 下载 `O2Cache_演示报告摘要.md`。

## 7. 如何生成二维码或运行网址

演示时可使用浏览器地址栏中的运行网址。若需要二维码，请先确认该地址在手机或其他设备上可以访问，再使用常见二维码工具生成。不要把 `localhost` 地址作为公网二维码提交。

## 8. 完整解说稿

逐页口播稿见 `docs/网页演示完整解说稿.md`。录制前建议先按该稿件走一遍完整流程，再根据视频时长删减个别句子。
