# 开发者说明

本文档用于保存不适合出现在评委演示界面的工程信息。用户界面只展示“应急智枢：面向灾害响应的算力动态优化与云边协同系统”的正式叙事。

## 与原实验工程的关系

当前 Streamlit 平台是在已有资源缓存预测实验工程基础上整理出的演示系统。平台复用了环境状态转移、非对称评价函数、历史案例库构建、相似案例检索、ICL 推理接口和结果导出等稳定模块，但没有重写核心训练或预测算法。

原始实验工程中的训练、评测和结果整理仍可通过命令行脚本独立执行。平台侧重点是把这些过程整理为可解释的可视化演示流程，包括任务负载接入、历史案例库构建、Top-K 相似案例检索、ICL 少样本缓存决策和反馈评价。

## 内部策略标识映射

用户界面不展示内部策略名。代码内部为兼容既有逻辑，仍保留部分历史标识：

| 内部标识 | 用户界面名称 | 说明 |
|---|---|---|
| `Baseline` | 即时规则分配基线 | 仅依据当前状态进行短视资源分配 |
| `Rule-Heuristic` | 启发式状态调节基线 | 根据请求趋势和排队压力进行规则修正 |
| `RAG-only` / `retriever` | 相似案例缓存预测（消融基线） | 使用 Top-K 历史案例动作融合，不调用外部 ICL 推理模型 |
| `RAG+LLM` / `llm` | ICL少样本缓存决策（主方法） | 将相似历史案例与当前状态组织为上下文输入，生成资源缓存量 |
| `MetaPPO` / `Meta-RL` | Meta-RL专家基线 | 可作为对比基线，也可提供初始历史决策轨迹 |

注意：本项目正式叙事不是通用文档问答系统。检索对象是资源调度历史案例，而不是文本知识库。

## 输入输出文件说明

演示数据默认放置在 `demo_data/`，包括任务负载样例和历史决策轨迹样例。运行结果默认整理到 `demo_outputs/`，用于本地演示和导出。

常见工程产物包括：

- `output.summary.json`：策略评估摘要。
- `output.trace.jsonl`：逐步决策过程。
- `output.trace.csv`：逐步决策表。
- `output.npz`：兼容原实验流程的数组结果。
- `policy_best.th`：原训练流程中的模型权重。
- `training_curves.json`：原训练流程中的曲线记录。

这些文件名不应作为用户界面的主要表述。用户界面应使用“运行结果”“策略评估摘要”“逐步决策结果”“历史案例表”等中文术语。

## 外部 ICL 推理模型接口

外部模型接口只作为高级配置存在，默认关闭。未配置访问密钥时，平台会使用本地相似案例缓存预测作为回退，不影响基本演示。

当前实现要求：

- API Key 只在当前会话中使用，不保存。
- 不在导出结果中写入访问密钥。
- 调用失败时回退到相似案例缓存预测。
- 不因外部服务不可用导致平台整体不可用。

## 尚未完全接入的在线案例库更新机制

论文和说明材料中的完整闭环包括“反馈评价驱动案例库动态更新”。当前平台已经展示了反馈评价、案例质量分析和更新条件分析，但在线运行后的新案例自动写入历史案例库尚未完全接入主流程。

因此，用户界面应表述为“反馈评价与案例库更新条件分析”或“展示反馈驱动更新机制”，不要宣称已经实现生产式在线动态入库。

后续扩展建议：

1. 将运行后的时隙记录构造成候选案例。
2. 根据综合评价值、请求保障率、资源冗余率和动态阈值判断是否入库。
3. 对案例库统计量使用 EMA 方式更新。
4. 为新增案例提供人工确认或演示模式开关。
5. 将更新后的案例库持久化为可复用的演示数据。
