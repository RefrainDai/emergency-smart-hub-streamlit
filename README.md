# 应急智枢：面向灾害响应的算力动态优化与云边协同系统

作者：北京邮电大学O₂Cache团队

## 平台简介

本平台是面向灾害响应多模态任务服务的资源预分配与调度决策支持工具。平台展示如何通过历史案例库、Top-K 相似案例检索和上下文学习机制，预测下一时隙资源缓存量，为模型实例预热、GPU/显存预留和并发资源分配提供辅助决策。

核心流程：

```text
灾害任务请求接入
→ 任务负载状态建模
→ 历史案例库构建
→ Top-K 相似案例检索
→ ICL 少样本缓存决策
→ 资源缓存量与资源预热建议
→ 反馈评价与更新条件分析
```

## 核心功能

- **系统总览**：展示平台定位、核心流程、关键能力和演示指标。
- **多模态任务接入**：以演示模式展示灾害图像与文本请求如何形成智能体工具调用轨迹，并转化为任务服务负载。
- **灾害任务接入**：接入多模态任务服务请求负载，展示任务类型、请求序列和请求到达量变化。
- **历史案例库**：构建由历史状态、请求窗口、缓存决策和反馈评价组成的案例库。
- **相似案例检索**：展示当前状态对应的 Top-K 相似历史案例及相似度来源。
- **ICL缓存决策**：生成下一时隙资源缓存量与资源预热建议。
- **调度效果评估**：对比请求保障率、资源冗余率和综合评价值。
- **单步可解释分析**：复盘某一时隙的状态、相似案例、决策来源和反馈结果。
- **结果导出**：导出运行结果、策略对比结果、单步决策解释、历史案例表和演示报告摘要。

## 快速开始

进入项目目录后安装依赖并启动：

```bash
pip install -r requirements.txt
streamlit run app.py
```

启动后浏览器通常会打开：

```text
http://localhost:8501
```

## Windows 启动方式

推荐方式：

```bat
scripts\run_windows.bat
```

也可以在 PowerShell 或命令提示符中运行：

```bat
pip install -r requirements.txt
streamlit run app.py
```

## macOS / Linux 启动方式

推荐方式：

```bash
bash scripts/run_unix.sh
```

也可以直接运行：

```bash
pip install -r requirements.txt
streamlit run app.py
```

## 推荐演示流程

1. 打开“系统总览”，介绍 O₂Cache、应急智枢和核心流程。
2. 打开“多模态任务接入”，展示智能体工具调用轨迹如何转化为任务服务负载。
3. 打开“灾害任务接入”，展示多模态任务服务请求负载。
4. 打开“历史案例库”，构建案例库并查看案例质量分布。
5. 打开“相似案例检索”，选择一个任务序列，展示 Top-K 相似历史案例。
6. 打开“ICL缓存决策”，运行 ICL 少样本缓存决策或相似案例缓存预测。
7. 打开“调度效果评估”，生成策略对比结果。
8. 打开“单步可解释分析”，说明某一时隙的决策依据。
9. 打开“结果导出”，生成演示报告摘要并导出相关表格。

## 常见问题

**1. 浏览器没有自动打开怎么办？**  
手动访问 `http://localhost:8501`。

**2. 提示找不到 streamlit 怎么办？**  
先运行 `pip install -r requirements.txt`，确认依赖安装完成后再启动。

**3. 页面提示尚未构建案例库怎么办？**  
进入“历史案例库”页面，选择内置历史决策轨迹并点击“构建/刷新历史案例库”。

**4. ICL 推理模型接口必须配置吗？**  
不必须。未配置外部接口时，平台会使用本地相似案例缓存预测完成演示。

**5. 导出的结果在哪里？**  
点击“结果导出”页面中的按钮后，浏览器会下载对应文件。

## 注意事项

- 演示结果基于当前加载的任务负载和当前运行配置生成。
- 反馈评价用于分析新案例是否具备入库价值，自动在线写入案例库可在后续版本继续扩展。
- 不要在平台目录中放置 `.env`、API Key、虚拟环境目录或无关大文件。
